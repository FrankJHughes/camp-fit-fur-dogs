# Story Template

## Story Grammar (Required)
All stories **must** begin with the following structure:

**As a <role>, I must/should be able to <verb>… so that <value>…**

Valid roles:
- Owner  
- Staff  
- Admin  

Invalid roles:
- System  
- API  
- Backend  
- Infrastructure  
- Database  

Stories must always reflect a **human actor**, never a technical component.

---

## Intent
_As a <role>, I must/should be able to <verb>… so that <value>…_

Describe the human need and the outcome they expect.

## Value
Explain why this story matters to the user and the business.

## Acceptance Criteria
- List observable, testable behaviors.
- Avoid implementation details.
- Avoid technical tasks disguised as stories.

## Emotional Guarantees
List the emotional guarantees this story must uphold (e.g., EG‑01 No Surprises, EG‑03 Calm Protection).

## Notes
Add architectural constraints, purity rules, or cross‑slice considerations.
