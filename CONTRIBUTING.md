# Contributing

Please read `docs/governance.md` before contributing.

## How to contribute
- Create a feature branch from `trunk`: `feature/<id>-short-name`.
- Open a PR referencing the issue or story.
- Keep PRs small and focused.

## PR checklist
- Code builds locally.
- Unit tests added/updated and passing.
- Docs updated when behavior changes.
- CI green and at least one approving review.

## Code style
- Follow repository EditorConfig and formatting rules.
