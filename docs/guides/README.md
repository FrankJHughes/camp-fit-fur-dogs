# Guides

Guides are **explanatory documents** that teach contributors how to work within the Camp Fit Fur Dogs system.

Guides are **not** runbooks (procedures) and **not** conventions (rules).
They provide context, reasoning, and walkthroughs for roles, workflows, and domain concepts.

---

## Purpose

Guides exist to:

- Explain how the system works
- Provide role-specific onboarding
- Clarify workflows and processes
- Offer conceptual understanding
- Support decision-making

Guides are **explanatory**, not normative.

For rules and standards, see:

- **../conventions/architecture.md**
- **../conventions/workflow.md**
- **../conventions/code.md**

For operational procedures, see:

- **../runbooks/README.md**

## When to Add a Guide

Create a guide when:

- A concept needs explanation
- A workflow needs clarification
- A role needs onboarding material
- A domain area needs a walkthrough
- A process is too conceptual for a runbook

Guides should be:

- Clear
- Conceptual
- Explanatory
- Example-driven
- Role-aware

---

## When *Not* to Add a Guide

Do **not** create a guide for:

- Operational steps → belongs in runbooks
- Rules or standards → belongs in conventions
- Architectural decisions → belongs in ADRs
- Sprint artifacts → belongs in sprint reviews
- Product intent → belongs in stories

Guides are for **understanding**, not **execution** or **rules**.
