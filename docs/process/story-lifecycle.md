# Story Lifecycle

## 1. Backlog Stage
- Story Markdown is created.
- Contains product intent, value, emotional safety, acceptance criteria.
- YAML is NOT required.

## 2. Sprint Planning Stage
- Story is added to sprint YAML.
- Tooling generates missing Story YAML.
- YAML becomes the planning artifact.

## 3. Sprint Commitment Stage
- YAML must exist.
- YAML must contain planning metadata.
- Sprint cannot be Ready without YAML.

## 4. Execution Stage
- YAML reflects planning state.
- Markdown remains the product source of truth.
