# Backlog Hygiene Rules

## Allowed
- Story YAML created early (optional).
- Story YAML created during sprint planning (preferred).
- Story Markdown created independently of YAML.

## Not Encouraged
- Auto-generating YAML at story-creation time.
- Treating YAML as required for backlog stories.
- Using YAML to store product-layer information.

## Required
- YAML must exist for any story included in an active sprint.
- YAML must include a sprint assignment once committed.
- YAML must pass readiness validation before sprint commitment.
