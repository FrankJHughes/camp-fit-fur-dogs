# Definition of Ready

## Story-Level DoR
A story is Ready when:
- It has clear value and intent.
- It has acceptance criteria.
- It has no unresolved dependencies.
- It is small enough for a sprint.
- It is understood by the team.

YAML is NOT required for a story to be Ready.

## Sprint-Level DoR
A sprint is Ready when:
- All included stories are Ready.
- All included stories have YAML.
- All planning metadata is present.
- Dependencies are resolved.
- Story points are assigned.

YAML IS required for the sprint to be Ready.
