# Session Cookie Conventions (CampFitFurDogs)

CampFitFurDogs owns session creation, session persistence, and session cookie
issuance. Frank does not participate in session management.

The session cookie is the primary authentication mechanism for Owners.

---

## Cookie Name

The session cookie must be named:

````text
cfd.session
````

---

## Cookie Value

The cookie value must:

- be opaque  
- contain no user data  
- contain no dots  
- not be a JWT  
- not be base64‑encoded user information  
- be generated by the Application callback builder  

The Api callback endpoint **must not** generate cookie values.

---

## Cookie Flags

The session cookie must be issued with:

- `HttpOnly = true`  
- `Secure = true` (preview/prod)  
- `SameSite = Lax`  
- `Path = /`  
- `Max‑Age` set  

---

## Cookie Issuance

Cookies must be issued:

- only by Api endpoints  
- only after successful authentication  
- only using the `CookieValue` computed by the Application callback builder  

Builders must not write to HttpContext.

---

## Prohibitions

Session cookies must not:

- contain identity provider tokens  
- contain PII  
- contain claims  
- contain timestamps  
- contain signatures  

Session cookies must not be:

- JWTs  
- encrypted blobs  
- serialized domain objects
